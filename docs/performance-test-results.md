## 8. 瓶颈分析与优化建议

### 发现的瓶颈

1. **数据库查询端点偏慢** (`/api/observations`, `/api/stats`): 10-25ms
   - 可能原因: 缺少索引或 N+1 查询
   - 建议: 检查 SQL 查询计划，添加复合索引

2. **批量写入性能**: 5ms/条表现良好，但可考虑批量提交优化

### 优化建议

1. **数据库层面**: 为 observations 表添加 `(agent_id, created_at)` 复合索引
2. **缓存层**: 对 `/api/stats` 等聚合查询添加短时缓存 (5-10s TTL)
3. **连接池**: 检查 HikariCP 连接池配置是否合理
4. **批量 API**: 考虑添加批量 Observation 创建端点 (一次提交多条)

### 优秀指标

- ✅ 并发处理能力强 (20并发无压力)
- ✅ 写入性能稳定 (线性扩展)
- ✅ 健康检查极快 (<1ms)
- ✅ JVM 内存占用合理 (~400MB)
